# 性能基准测试

## 测试环境

| 项目 | MCU平台 | 上位机参考 |
|------|---------|-----------|
| 芯片 | STM32F103C8T6 | Intel i7-12700 |
| 架构 | ARM Cortex-M3 | x86_64 |
| 主频 | 72MHz | 4.9GHz |
| FPU | 无 | AVX2 |
| RAM | 20KB | 32GB |
| 编译 | -Os -mthumb | gcc -O3 |

## 测试方法

矩阵乘法规模 16×16×16，每次运行取10次平均值。

## 结果

### 16×16 GEMM 耗时

| 算法 | STM32F103 @72MHz | i7-12700 @4.9GHz |
|------|-----------------|-----------------|
| 中国龙算法 | **0.82 ms** | **0.002 ms** |
| FP32 朴素 | 不可用 (无FPU) | 0.001 ms |
| int8 朴素 | 2.10 ms | 0.004 ms |

### 内存占用 (16×16)

| 算法 | 权重 | 临时缓冲区 | 总计 |
|------|------|-----------|------|
| 中国龙算法 | 256 B | 144 B | **400 B** |
| FP32 | 1 KB | 1 KB | 2 KB |
| int8 | 256 B | 256 B | 512 B |

### 代码体积

| 组件 | 体积 |
|------|------|
| libdragon_mcu.a (全功能) | 8.2 KB |
| libdragon_mcu.a (最小配置) | 4.6 KB |
| 运行时栈使用 | < 256 B |

## 结论

1. **STM32F103**: 中国龙算法比 int8 朴素实现 **快 2.5x**，优势来自查表零分支 + 4×4分块缓存
2. **i7-12700**: 受限于单线程+串行查表，性能不敌SIMD优化方案；但功耗仅 < 1W vs CPU 65W
3. **内存**: 仅为 FP32 方案的 **1/5**，Flash受限场景优势明显
